The main type used for communication with the FTP server.

Once calling the constructor a connection to the specified socket address is opened, allowing for login and additional operations to be performed.

Example:

Connection

use ftp_lib::FtpConnection;
use std::net::SocketAddrV4;

let mut ftp_conn = FtpConnection::connect(
    "127.0.0.1:21".parse().unwrap()
).unwrap(); // Initiate the connection

ftp_conn.quit().unwrap();

Initiates the connection to the FTP server.

After initiating the stream of data to the server, the function waits for a response telling it if the service is available or if the service is not yet ready. If and once the server is ready, the function will return a functional instance of FtpConnection ready for varius operations to be performed.

pub fn login(
    &mut self,
    username: &str,
    password: Option<&str>
) -> Result<(), FtpError>[src]

Authenticates the connection to the FTP server.

When connecting to an FTP server the first command expected is the USER command specifying a username. From there a password should then be given logging the user in if their credentials were correct. A failure to authenticate will return an error of the type FtpError::NotLoggedIn.

Example:

Login to 127.0.0.1 then close the connection.

use ftp_lib::FtpConnection;
use std::net::SocketAddrV4;

let mut ftp_conn = FtpConnection::connect(
    "127.0.0.1:21".parse().unwrap()
).unwrap();

// login(username, password)
ftp_conn.login("anonymous", Some("fake@email.service")).unwrap(); // Login to the server

ftp_conn.quit().unwrap();

pub fn quit(&mut self) -> Result<(), FtpError>[src]

Terminates the connection to the FTP server.

As long as no connection errors occur while sending the quit command or shutting down both read and write interfaces, the connection will be severed, despite what the servers response is.

Example:

Connect to localhost then close the connection.

use ftp_lib::FtpConnection;
use std::net::SocketAddrV4;

let mut ftp_conn = FtpConnection::connect(
    "127.0.0.1:21".parse().unwrap()
).unwrap();

ftp_conn.login("anonymous", Some("fake@email.service")).unwrap();

// Preform actions with the FTP server

ftp_conn.quit().unwrap(); // End the connection to the server.

pub fn cd(&mut self, path: &str) -> Result<(), FtpError>[src]

Changes the current working directory in the FTP server.

Works similarly to the cd command in shells. On successful completion of this command, pwd should often follow to be absolutely sure the program knows where it currently is, this is due to the fact that the client cannot be certain of how the server will behave in response.

Example:

Connect to 127.0.0.1 then change to the /test_dir directory.

use ftp_lib::FtpConnection;
use std::net::SocketAddrV4;

let mut ftp_conn = FtpConnection::connect(
    "127.0.0.1:21".parse().unwrap()
).unwrap();

ftp_conn.login("anonymous", Some("fake@email.service")).unwrap();

ftp_conn.cd("test_dir").unwrap(); // Change the working directory to rfc

ftp_conn.quit().unwrap();

pub fn cdup(&mut self) -> Result<(), FtpError>[src]

Changes the current working directory in the FTP server to the parent directory.

Simmilar to the cd function, but simply goes up to the parent directory. This function is the same as cd .. in most shells.

Example:

Connect to localhost then go into /test_dir and return to /.

use ftp_lib::FtpConnection;
use std::net::SocketAddrV4;

let mut ftp_conn = FtpConnection::connect(
    "127.0.0.1:21".parse().unwrap()
).unwrap();

ftp_conn.login("anonymous", Some("fake@email.service")).unwrap();

ftp_conn.cd("test_dir").unwrap();
println!("PWD: {:?}", ftp_conn.pwd());

ftp_conn.cdup().unwrap(); // Return to / from /test_dir
println!("PWD: {:?}", ftp_conn.pwd());

ftp_conn.quit().unwrap();
